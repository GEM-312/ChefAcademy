# UX_REVIEW_2026-05-25.md
## Weekly UX Audit — Pip's Kitchen Garden
**Date:** 2026-05-25 | **Reviewer:** Claude (automated sweep) | **Doc-drift check:** Exit 0 — all green

## Section 1 — Summary

The app's core grow → cook → feed loop reads cleanly and the botanical watercolor aesthetic is well-maintained. Twelve anti-patterns (A–L) were evaluated across all 87 Swift source files. Three P0 blockers require immediate attention before the next playtest: bug-rescue emoji tap targets are fingernail-sized for 6yr olds, the seed carousel has no trailing-fade scroll cue, and the harvest mechanic's "Drag Pip" affordance lives only in small subtitle text. Fourteen P1 friction items and eight P2 polish items follow. Bright spots include the `PlantingSheet` morph pattern, `FarmShopView`'s two-tap confirm-before-spend dialog, and consistent `.trailingFade()` on profile, recipe, shop, and dashboard carousels.

## Section 2 — P0 Blockers

### P0-01 — Bug-rescue tap targets are 10–12pt emoji (Anti-pattern D)
**File:** `ChefAcademy/PlotView.swift` — `bugRescuePlot` (approx. lines 340–390)
**Anti-pattern:** D (touch target below 44pt)

Three 🐛 bug emoji are rendered at `.font(.system(size: 12))`, `.font(.system(size: 10))`, and `.font(.system(size: 11))` inside small positional `ZStack`s. Each has `.onTapGesture` but no `contentShape` expansion. A 6yr old's fingertip (roughly 44–48pt across) entirely covers the character — accurate tapping is luck, not skill. Same issue for weed emoji in `weedingPlot` (`🌿` at 10–14pt), though weeds use a swipe gesture rather than tap so the target is less critical.

**Fix:**
Wrap each bug in a minimum 44pt tappable area. Replace the emoji with a `Text("🐛")` inside a `ZStack` with an expanded `contentShape`:

```swift
// Per bug in bugRescuePlot ForEach:
ZStack {
    // transparent hit expansion
    Color.clear
        .frame(width: AppSpacing.minTapTarget, height: AppSpacing.minTapTarget)
    Text("🐛")
        .font(.AppTheme.rounded(size: 28, weight: .regular))  // 28pt visible, 44pt hit
}
.contentShape(Rectangle())
.onTapGesture { rescueBug(at: index) }
```

Increase weed emoji to `.font(.AppTheme.rounded(size: 22, weight: .regular))` minimum so kids can actually see and interact with them.

### P0-02 — Seed inventory carousel has no trailing-fade scroll cue (Anti-pattern J / C)
**File:** `ChefAcademy/GardenView.swift` — `seedInventorySection`, approx. line 1032
**Anti-pattern:** J (carousel without trailing fade), C (no scroll affordance)

The seed bag `ScrollView(.horizontal, showsIndicators: false)` renders seeds side by side with no indicator that more exist. iOS hides the scrollbar until the gesture starts. A 6yr old who sees 3 bags and doesn't see a fade will assume those are all their seeds and miss everything past the fold. Every other horizontal carousel in the app (`RecipeListView` categories, `FarmShopView` categories, `ProfilePickerView` profiles, `ParentDashboardView` child tabs) correctly applies `.trailingFade()` — this one was missed.

**Fix:**
```swift
// In seedInventorySection, on the horizontal ScrollView:
ScrollView(.horizontal, showsIndicators: false) {
    HStack(spacing: AppSpacing.sm) {
        ForEach(gameState.seeds, id: \.id) { seed in
            SeedBadge(seed: seed) { ... }
        }
    }
    .padding(.horizontal, AppSpacing.md)
}
.trailingFade()  // ← add this
```

### P0-03 — Harvest "Drag Pip" mechanic has no persistent visual affordance (Anti-pattern K)
**File:** `ChefAcademy/GardenView.swift` — `PipGardenMessage` / `gardenHint` subtitle
**Anti-pattern:** K (gesture discovery)

The harvest mechanic requires the child to drag Pip character onto a ready plot. The only instruction is a small subtitle `gardenHint` text inside the `PipGardenMessage` bubble, shown only when a plot is ready. There is no persistent visual cue — no arrows, no animation showing Pip can be moved, no animated drag shadow on Pip himself. The `DraggablePipView` updates position but the child has zero indication Pip is draggable unless they happen to read the small hint text (which a 6yr old often won't).

**Fix:**
Add a gentle looping bounce/wiggle on the Pip drag handle when any plot is in the `ready` state, and a persistent drag-arrow badge under Pip. Use existing tokens:

```swift
// In DraggablePipView body, conditionally when anyPlotReady:
PipWavingAnimatedView(size: .custom(pipSize))
    .overlay(alignment: .bottom) {
        if anyPlotReady {
            Image(systemName: "arrow.up.and.down.circle.fill")
                .font(.AppTheme.rounded(size: 22, weight: .medium))
                .foregroundColor(Color.AppTheme.goldenWheat)
                .offset(y: 12)
                .transition(.scale.combined(with: .opacity))
        }
    }
    .scaleEffect(anyPlotReady ? 1.05 : 1.0)
    .animation(AnimationConstants.floatLoop, value: anyPlotReady)
```

## Section 3 — P1 Friction

### P1-01 — SeedBadge has no tappable visual affordance (Anti-pattern C)
**File:** `ChefAcademy/GardenView.swift` — `SeedBadge` view
**Anti-pattern:** C (no tappability cue)

`SeedBadge` uses `.onTapGesture` but looks like static content — no border, no shadow, no scale feedback, no `.buttonStyle`. Kids who touch it get no "I pressed something" confirmation and may give up. The seed bag image renders flat on the carousel with zero affordance.

**Fix:** Wrap in a `Button` and apply `BouncyButtonStyle`:
```swift
Button { onTap() } label: { seedBagContent }
.buttonStyle(BouncyButtonStyle())
```

### P1-02 — CookingCompletionView delays action buttons behind a 2s+ animation (Anti-pattern H)
**File:** `ChefAcademy/CookingCompletionView.swift` — star/reward reveal sequence
**Anti-pattern:** H (reward screen delay without CTA)

Stars pop in one by one at 0.3s intervals, then reward chips appear, then the health boost card, then action buttons. Total wait before a button exists: approximately 2.2–2.5s. During this gap the screen is visually active but offers no tap target. A 6yr old will tap/swipe randomly, potentially triggering unintended navigation or feeling ignored.

**Fix:** Show a "Tap to continue" pulse affordance immediately (below the star row), or make the action buttons visible-but-dimmed from the start and animate them to full opacity once stars finish:
```swift
// Bottom section always rendered, enabled after animation completes
HStack {
    Button("Back to Kitchen") { ... }
        .buttonStyle(TexturedButtonStyle(tint: Color.AppTheme.parchment))
    Button("See how your food helps!") { ... }
        .buttonStyle(TexturedButtonStyle(tint: Color.AppTheme.goldenWheat))
}
.opacity(starsRevealed ? 1.0 : 0.0)
.animation(AnimationConstants.fadeMedium, value: starsRevealed)
```

### P1-03 — Watering mechanic uses Timer.scheduledTimer for progress (Architecture + UX)
**File:** `ChefAcademy/PlotView.swift` — `wateringPlot` / `WaterPourCharacterView` (approx. line 428)
**Anti-pattern:** E (confusing state without feedback) + architecture violation

`wateringPlot` uses `Timer.scheduledTimer` directly for the progress ring animation. Per CLAUDE.md Architecture Rule 2: "Game physics loops use `TimelineView(.animation)` with delta-time, NOT `Timer.scheduledTimer` at 60fps." Timer-based physics ties speed to the RunLoop; ProMotion 120Hz devices and CPU throttling can break it. The drain-on-lift-finger behavior also lacks a visual "success" moment.

**Fix:** Port the progress ring to `TimelineView(.animation)` with a delta-time accumulator matching the pattern already in `InsulinTetrisView` and `WaterPourCharacterView`.

### P1-04 — Kitchen 4-phase flow requires sequential invisible hotspot taps (Anti-pattern B)
**File:** `ChefAcademy/KitchenView.swift` — 4 cooking phases
**Anti-pattern:** B (multi-step invisible flow)

The cooking flow has 4 phases: browsing → gatherPantry → moveToStove → stoveReady. Each phase requires tapping a specific invisible button hotspot overlaid on the kitchen background image (Pantry, Counter, Stove spots). A child who taps the wrong area in the kitchen image receives no feedback. Pip's floating bubble tells them what to tap textually, but the hotspots themselves are invisible `ZStack` `Button`s with no visual ring, glow, or pulse.

**Fix:** Add a pulsing highlight ring on the active spot for the current phase:
```swift
// On each KitchenSpotButton when it is the active target:
if isActiveTarget {
    Circle()
        .stroke(Color.AppTheme.goldenWheat.opacity(0.8), lineWidth: 3)
        .scaleEffect(pulseScale)  // @State animated with AnimationConstants.floatLoop
        .frame(width: spotSize + 16, height: spotSize + 16)
}
```

### P1-05 — Nutrition fact pills in RecipeDetailView are text-only, no emoji (Anti-pattern F)
**File:** `ChefAcademy/RecipeDetailView.swift` — nutrition facts `FlowLayout` (approx. lines 258–270)
**Anti-pattern:** F (adult vocabulary / literacy dependency)

Nutrition fact pills render raw strings like `"Protein"`, `"Vitamin C"`, `"Calcium"` — pure text labels with no emoji or icon. `NutrientType.emoji` already maps each nutrient to an emoji in `SeedInfoView.swift`. Kids who can't read will see a row of colored blobs with mystery words. The emoji mapping exists — it's just not wired here.

**Fix:** Map recipe nutritionFacts strings to `NutrientType` where possible and prepend the emoji:
```swift
// In FlowLayout content for nutritionFacts:
Text("\(nutrientEmoji(fact)) \(fact)")
    .font(.AppTheme.caption)
    .foregroundColor(Color.AppTheme.darkBrown)
    .padding(.horizontal, 12)
    .padding(.vertical, 6)
    .background(Color.AppTheme.sage.opacity(0.15))
    .cornerRadius(14)
```
Or add emoji directly to the `Recipe.nutritionFacts` string array at the data layer.

### P1-10 — AssembleMiniGame has both Button AND onTapGesture — double-trigger risk
**File:** `ChefAcademy/CookingMiniGames.swift` — `AssembleMiniGame.body`
**Anti-pattern:** E (confusing state)

`AssembleMiniGame` applies `.onTapGesture` to the whole `body` container AND contains a `Button` inside. Tapping the button triggers both the button's action AND the tap gesture on the parent, which can fire the completion handler twice. The second fire likely calls `onComplete(score)` on an already-completed mini-game.

**Fix:** Remove the `.onTapGesture` from the container and route all interactions through the `Button` inside, or add a guard `guard !isComplete else { return }` at the top of the onTap closure.

### P1-13 — PipShopMessage reshuffles message on every re-render (Anti-pattern E)
**File:** `ChefAcademy/FarmShopView.swift` — `PipShopMessage` body
**Anti-pattern:** E (confusing state)

`PipShopMessage` calls `shopMessages.randomElement()` directly inside `body` with no `@State` capture. SwiftUI can re-invoke `body` on any parent state change (scroll offset, item selection, etc.), reshuffling Pip's message mid-sentence. Kids following along aurally will hear a different Pip message suddenly appear.

**Fix:**
```swift
struct PipShopMessage: View {
    @State private var message: String = ""

    var body: some View {
        PipSpeechBubble(message: message)
            .onAppear {
                message = shopMessages.randomElement() ?? shopMessages[0]
            }
    }
}
```

## Section 6 — Pattern Wins

These deserve recognition as correctly implemented patterns:

- **`.trailingFade()` coverage:** `RecipeListView`, `FarmShopView` (categories + pantry), `ProfilePickerView`, `ParentDashboardView` child tabs — all correct. Only garden seed carousel and (possibly) sibling carousel are missing it.
- **`PlantingSheet` morph pattern:** `matchedGeometryEffect` seed→NPC-detail morph is executed correctly — seed card morphs to full NPC detail with "Plant" CTA always visible. No scrolling required to find the primary action. Exemplary for the age 6+ target.
- **`FarmShopView` two-tap confirm:** Shop item tap → info sheet → "Buy" button → `PipDialogView` confirm overlay. Kids can explore without accidentally spending coins. Anti-pattern A is fully resolved here.
- **`TimelineView` physics in production games:** `HealthyChoiceGameView`, `InsulinTetrisView`, and `WaterPourCharacterView` all use `TimelineView(.animation)` with delta-time accumulators — correct per architecture rule 2.
- **`PipVoice` double-speak prevention:** `PipSpeechBubble` and `PipHeaderStack` are the only callers of `PipVoice.shared.speak(...)` — no manual `speak()` calls found adjacent to these components in the active codebase.
- **`PINPadGrid` consolidation:** `ParentPINEntryView`, `FamilySetupView`, and `MigrationPINSetupView` all use the shared `PINPadGrid` from `PipComponents.swift` — no local copies reintroduced.
- **`FamilyMeetPipStep` 3-dialog flow:** Correctly trimmed from 6 dialogs to 3 per prior audit. "Let's grow!" CTA surfaces on dialog 3 — kids reach the action in 3 taps.
- **Drag gesture expanded hit boxes:** `AddToPanMiniGame` and `InsulinTetrisView.handleDrop` both expand bin hit zones by +20–40pt for forgiving kid-friendly interaction.
- **Allergen warnings:** Red terracotta banner in `RecipeDetailView` and allergen badge in `RecipeCardView` are both prominent and color-accurate.
- **`GlucoseJourneyView` snack quiz:** 3-option snack quiz with sugar-cube visual metaphor is age-appropriate and self-explanatory — no literacy required to understand "6 sugar cubes vs 1 sugar cube."
