## FOCUS 1 — Stale UI State Bugs

### F1-A: `DispatchQueue.main.async` — BANNED (17 callsites, 6 files)

All must be replaced with `Task { @MainActor in }`. If the mutation is delayed, add `try? await Task.sleep(for: .seconds(X))` inside the Task and assign to a cancellable `@State var task: Task<Void, Never>?`.

| File | Approx lines | Context |
|---|---|---|
| `MultiplayerManager.swift` | ~65, ~196, ~241, ~297, ~321 | GKMatch delegate callbacks mutating @Published |
| `NearbyMultiplayerManager.swift` | ~155, ~197, ~221, ~241, ~287, ~307 | MCSession delegate callbacks mutating @Published |
| `AuthManager.swift` | ~113 | Auth state mutation after background callback |
| `GameCenterMatchmakerView.swift` | 3 callsites | GKMatchmakerViewController delegate mutating @Published |
| `GameCenterService.swift` | authenticate() | GKLocalPlayer.authenticateHandler closure |
| `ParentPINEntryView.swift` | ~134 | `startAppleIDVerification()` — sets `isResettingPIN = true` |

**Pattern to apply:**
```swift
// BEFORE (banned)
DispatchQueue.main.async { self.someState = value }

// AFTER (correct)
Task { @MainActor in self.someState = value }
```

**Note for SeedInfoView.swift:** `VeggieCanvasView.updateUIView` uses `DispatchQueue.main.async { clearToggle = false }` inside a UIViewRepresentable bridge. This is a borderline case — `updateUIView` already runs on main, so the dispatch is redundant but not dangerous. Convert to a direct assignment or `Task { @MainActor in clearToggle = false }` to satisfy the letter of the rule.

## Refactor / Architecture Suggestions

**R-01: PIN UI — eliminate three redundant local copies.**
`MigrationPINSetupView.swift`, `ParentPINEntryView.swift`, and `FamilySetupView.swift` each have hand-coded PIN dot rows with hardcoded `frame(width: 20, height: 20)` and `lineWidth: 1`. CLAUDE.md rule 4 explicitly states: "Use the shared `PINPadGrid<Leading, Trailing>` and `PINButton` from `PipComponents.swift`. Three views previously had local copies — never reintroduce." This has regressed. Audit these three files and migrate to the shared component.

**R-02: WeatherOverlayView seasonal particle loops — extract `AnimationConstants`.**
`SeasonalOverlayView.startParticleAnimation()` and the five weather overlays collectively define ~10 hardcoded loop animations. Add a `WeatherAnimations` namespace to `AnimationConstants` (or extend the existing struct) with `sunPulse`, `cloudDrift`, `windStreak`, `seasonalDrift`, `seasonalPulse`. Then each overlay becomes a one-liner referencing the constant.

**R-03: Gender avatar routing — one global grep before release.**
Run `grep -r "boy_card_clean\|girl_card_clean" ChefAcademy/ --include="*.swift"` to find any remaining inline ternaries not caught in this review. The parent `mom/dad` frame routing in `UserProfile.profilePoseImage` is correct; the child bypass in 8 files listed under F2-D defeats it.

**R-04: `Spacer().frame(height: N)` — resolve ambiguity.**
Several files use `Spacer().frame(height: 80)` where the intent is tab-bar clearance (100pt). Establish a firm rule: anything providing clearance for the floating tab bar must use `AppSpacing.tabBarClearance` (100). Anything providing internal rhythm spacing between sections must use a real `AppSpacing.*` token. The 80pt variants in `SiblingProfileView`, `PlayLearnView`, `BodyBuddyView` are likely under-shooting the tab bar — audit each one.
