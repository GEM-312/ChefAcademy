## Section 3 — P0 Critical Violations

### P0-A: `try? context.save()` — Silent Data Loss (13 sites, 8 files)

**Rule:** CLAUDE.md §1 — "Use `do { try save() } catch { print(error) }` for SwiftData saves — never `try?`. Silent failures destroyed child profiles for a week (March bug)."

This rule was written *because* of a real incident. None of the sites below have been fixed.

| File | Lines |
|------|-------|
| `SessionManager.swift` | 100, 206, 355, 419 |
| `FamilySetupView.swift` | 204 |
| `ParentDashboardView.swift` | 461, 490 |
| `AllergenEditorSheet.swift` | 119 |
| `ChefAcademyApp.swift` | 464 |
| `SiblingProfileView.swift` | 275 |
| `SiblingGardenView.swift` | 36, 176 |
| `AddChildFlowView.swift` | 132 |

**Fix pattern** (same for every site):
```swift
// BEFORE (silent failure)
try? modelContext.save()

// AFTER (logs the error so it's diagnosable)
do {
    try modelContext.save()
} catch {
    print("[Save] \(#function): \(error)")
}
```

SessionManager.swift is highest risk (4 sites) — it's the profile CRUD coordinator. Failure here is invisible to the user and corrupts family data silently.

### P0-B: `DispatchQueue.main.async` in Non-Bridge Code (13 sites)

**Rule:** CLAUDE.md §2 — "ZERO inline `DispatchQueue.main.asyncAfter` in `ChefAcademy/`. Every delayed UI mutation goes through `Task { @MainActor in }`."

The script confirms zero `asyncAfter` calls — the first half of the rule holds. But 11 `DispatchQueue.main.async` calls remain in the two multiplayer manager classes, plus 2 in view-layer files.

**Multiplayer managers (clear violations):**

| File | Lines |
|------|-------|
| `MultiplayerManager.swift` | 65, 196, 242, 297, 321 |
| `NearbyMultiplayerManager.swift` | 155, 200, 221, 241, 286, 308 |

**View-layer violations:**

| File | Line | Context |
|------|------|---------|
| `SeedInfoView.swift` | 222 | State mutation in a view — no UIKit bridge |
| `ParentPINEntryView.swift` | 134 | PIN entry feedback — no UIKit bridge |

**Grey areas (UIKit bridge pattern — acceptable):**
- `AuthManager.swift:113` — Sign in with Apple callback
- `GameCenterService.swift:102` — GKLocalPlayer.authenticateHandler (UIKit callback)
- `GameCenterMatchmakerView.swift:42, 50, 59` — UIViewControllerRepresentable coordinator

**Fix pattern** for the manager classes:
```swift
// BEFORE
DispatchQueue.main.async { [weak self] in
    self?.somePublishedProperty = value
}

// AFTER
Task { @MainActor [weak self] in
    self?.somePublishedProperty = value
}
```

### P0-C: Countdown `Timer.scheduledTimer(repeats: true)` (3 sites)

**Rule:** CLAUDE.md §2 — "Timer callbacks must wrap state mutations in `Task { @MainActor in }`. Game physics loops use `TimelineView(.animation)` with delta-time, not `Timer.scheduledTimer`."

One-shot spawn timers (`repeats: false`) wrapping `Task { @MainActor in }` are an accepted grey area. Repeating countdown timers are a direct violation — they create a RunLoop dependency and produce data-race warnings under strict concurrency.

| File | Line | What it does |
|------|------|--------------|
| `MultiplayerManager.swift` | 240 | 3-2-1 match countdown |
| `NearbyMultiplayerManager.swift` | 198 | 3-2-1 match countdown |
| `SplitScreenVersusView.swift` | 579 | 3-2-1 split-screen countdown |

**Fix pattern** (same for all three):
```swift
// BEFORE
Timer.scheduledTimer(withTimeInterval: 1.0, repeats: true) { [weak self] timer in
    Task { @MainActor in
        self?.countdownValue -= 1
        if self?.countdownValue <= 0 { timer.invalidate() }
    }
}

// AFTER
countdownTask = Task { @MainActor in
    while countdownValue > 0 {
        try? await Task.sleep(for: .seconds(1))
        guard !Task.isCancelled else { return }
        countdownValue -= 1
    }
    startMatch()
}
```

Assign to a `@State var countdownTask: Task<Void, Never>?`; cancel it on `.onDisappear`.

## Section 7 — Suggested Fixes (Existing Tokens Only)

All fixes below use **only existing** `Color.AppTheme.*`, `AppSpacing.*`, `AnimationConstants.*`, and `Font.AppTheme.*` tokens. No new hardcoded values.

**Fix C — Silent saves (highest urgency, affects user data):**
```swift
// In SessionManager.swift:100, 206, 355, 419 and all other sites:

    try context.save()        // or modelContext.save()
} catch {
    print("[Save] \(#function): \(error)")
```

**Fix A — DispatchQueue.main.async in managers:**
```swift
// Replace all DispatchQueue.main.async { [weak self] in ... } with:
Task { @MainActor [weak self] in
    guard let self else { return }
    // mutations here
}
```

**Fix B — Countdown timers:**
```swift
// Replace Timer.scheduledTimer(repeats: true) countdown with:
@State private var countdownTask: Task<Void, Never>?

private func startCountdown() {
    countdownTask = Task { @MainActor in
        var count = 3
        while count > 0 {
            guard !Task.isCancelled else { return }
            countdownValue = count
            try? await Task.sleep(for: .seconds(1))
            count -= 1
        }
        countdownValue = 0
        startMatch()
    }
}
// Cancel in .onDisappear { countdownTask?.cancel() }
```

**Fix D — profilePoseImage bypass:**
```swift
// BEFORE (in LocalVersusView, SplitScreenVersusView, etc.)
```

## Section 8 — What's Working Well

**Design tokens strongly adopted in main game views.** Garden, Kitchen, FarmShop, BodyBuddy, PlayLearn, CookingSession, RecipeDetail, CookingCompletion all use `Color.AppTheme.*`, `AppSpacing.*`, and `Font.AppTheme.*` correctly throughout. The botanical palette is coherent.

**No `.font(.system(size:))` in user-facing code.** `SceneEditor.swift` uses it (dev-only, exempt). Clean everywhere else.

**`PipSpeechBubble` / `PipHeaderStack` used correctly.** No double-speak pattern (manual `PipVoice.shared.speak()` adjacent to these auto-speaking components) found in any file.

**PipAIService SSE streaming is well-structured.** The two-hop tool-use loop, App Attest auth gating, rate-limit handling, and MainActor isolation all follow the pattern correctly.

**`SeededRandomGenerator` is deterministic.** xorshift64 with explicit seed — lockstep multiplayer won't desync from RNG differences.
