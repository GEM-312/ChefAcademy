## Section 1 — P0: Critical (Blocks the core GROW → COOK → FEED loop for age 6+)

### P0-1 · GardenView — Broken harvest instruction
**File:** `GardenView.swift` ~L860–862 (Pip removed from map), L1225 (hint text)  
The persistent `gardenHint` reads *"Drag Pip to a glowing plot to harvest!"* but Pip was removed from the map overlay at L860–862. Harvest is now triggered by tapping the plot directly.  
**Impact:** A 6-year-old who cannot find Pip will stall on the GROW step's completion action indefinitely. Misleading instruction for a core mechanic.  
**Fix direction:** Update hint copy to match the actual gesture ("Tap a glowing plot to harvest!") — or restore Pip-drag as the harvest trigger.

### P0-2 · PlotView — Bug rescue tap targets 12–16pt
**File:** `PlotView.swift` L383–392  
Bug emojis rendered via `Text("🐛").font(.AppTheme.rounded(size: [12, 10, 11][i]))`. Effective hit area ≈ 12–16pt. Apple HIG minimum is 44pt; age-6+ recommendation is 60pt.  
**Impact:** Kids cannot reliably tap squirming bugs → plant care stalls → GROW loop breaks.

### P0-3 · PlotView — Weed tap targets 10–14pt
**File:** `PlotView.swift` L311–318  
Weed emojis via `Text("🌿").font(.AppTheme.rounded(size: [14, 12, 10][i]))`. Same tap-target problem.  
**Impact:** Same as P0-2.

## Section 2 — P1: Important UX Issues

### P1-1 · PlotView — Action instructions unreadable (~10pt text)
**File:** `PlotView.swift` (overlay instruction Text views)  
"Hold me!", "Swipe up!", "Watering..." labels render at ~10pt. A 6-year-old cannot read these. Instructions that explain the required gesture are invisible.

### P1-2 · PlotView — Three undifferentiated gesture paradigms
**File:** `PlotView.swift`  
Water = hold/DragGesture; weed = swipe-up; bug = tap-per-emoji. All three share similar visual containers with no distinct affordance per gesture type. Kids must discover each independently with no consistent signal.

### P1-3 · GardenView — SeedBadge lacks button affordance
**File:** `GardenView.swift`  
`SeedBadge` uses `.onTapGesture` only — no `Button`, no press-highlight, no confirmation feedback on tap. Kids cannot tell whether their tap registered.

### P1-4 · KitchenView — "Farm Shop" badge not tappable
**File:** `KitchenView.swift` (almostReadySection)  
The "Farm Shop" pill badge shown when a recipe is missing ingredients is a `Text` view with no navigation action. It looks like a CTA and does nothing. Breaks the COOK path's redirect to the Shop.

### P1-5 · FarmShopView — Walk animation blocks shop on every visit
**File:** `FarmShopView.swift` (FarmTransitionView)  
Pip's walk-to-barn animation (~4–5 s) plays on every visit to the Shop tab. A child redirected from the Kitchen to buy a missing ingredient is forced to watch the full animation each time.  
**Fix direction:** Add a session-state flag so the animation plays only on first visit per session; subsequent entries skip straight to the shop.

### P1-6 · FarmShopView — Pantry scroll missing `.trailingFade()`
**File:** `FarmShopView.swift`  
Pantry items sit in a horizontal `ScrollView` without `.trailingFade()`. Per §4, `.trailingFade()` is mandatory for horizontal carousels. Kids cannot see that more items exist off-screen; pantry discoverability is broken.

### P1-7 · CookingSessionView — Dismiss button 24pt, no skip option
**File:** `CookingSessionView.swift`  
The "×" dismiss button is ~24pt with no explicit frame. There is no skip/exit option for a stuck mini-game step. A kid who cannot complete a step has no escape route with a reachable button.

### P1-8 · RecipeListView — Greeting hardcoded, not personalised
**File:** `RecipeCardExample.swift` (RecipeListView header)  
Header reads `"Hello, Little Chef! 👋"` regardless of who is playing. `SessionManager.currentProfile.name` is available. Personalisation matters strongly to the target audience.

### P1-9 · BodyBuddyView — "Cook More!" CTA wrong button style
**File:** `BodyBuddyView.swift`  
"Cook More!" uses `.buttonStyle(.plain)` + manual `.background() + .cornerRadius() + .shadow()` — violates §4. It lacks the bouncy press feedback every other primary CTA delivers, so kids get no tactile confirmation their tap worked.

### P1-10 · HomeAnimated — Today's recipe card uses `.onTapGesture`, no affordance
**File:** `HomeAnimated.swift` (todaysRecipeSection)  
`RecipeCardView` is tapped via `.onTapGesture { selectedRecipe = todaysRecipe }`, not a `Button`. No visual press feedback. For the Home screen's primary action (cook today's recipe), this is a serious discoverability loss.
