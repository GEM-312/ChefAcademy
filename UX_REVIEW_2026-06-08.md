## Section 3 — P0 Blockers

### P0-1 — Kid UX: Sub-44pt tap targets (UNRESOLVED from 2026-06-01)

**PlotView.swift ~line 389 — bugRescuePlot:**
Three ladybug emoji `Text` elements inside a `ZStack` with `.onTapGesture`. No `Button` wrapper, font sizes 10–12pt. These are the primary mechanic for the bug-rescue interaction and are functionally untappable for small fingers.
```swift
// Current (wrong):
Text("🐛").font(.AppTheme.rounded(size: [12, 10, 11][i])).onTapGesture { ... }

// Fix: wrap each in a Button with .frame(minWidth: 44, minHeight: 44)
Button { rescueBug(at: i) } label {
    Text("🐛").font(.AppTheme.rounded(size: 30))
}
.frame(minWidth: AppSpacing.minTapTarget, minHeight: AppSpacing.minTapTarget)
.buttonStyle(BouncyButtonStyle())
```

**GardenView.swift — DraggablePipView "Drag me!" hint (UNRESOLVED from 2026-06-01):**
The gesture affordance label renders at `Font.AppTheme.rounded(size: 9, weight: .semibold)` — 9pt is the smallest text in the codebase and is invisible at arm's length. The hint itself doesn't need a tap target, but the label needs to be at minimum `.caption` (12pt) and ideally `Font.AppTheme.body`.

## Section 7 — Suggested Redesigns

### SR-1: Sweep the auth/multiplayer DispatchQueue.main.async layer in one PR

All 16 Pattern C (new cycle) violations are the same mechanical fix. Rather than fixing one file at a time, a single focused PR across `MultiplayerManager`, `NearbyMultiplayerManager`, `GameCenterService`, and `GameCenterMatchmakerView` eliminates the cluster cleanly and makes the commit easy to bisect.

The replacement pattern is identical in every case:
```swift
// Every GKMatchDelegate / MCSessionDelegate / GKMatchmakerViewControllerDelegate callback:
func match(_ match: GKMatch, player: GKPlayer, didChange state: GKPlayerConnectionState) {
    Task { @MainActor [weak self] in
        guard let self else { return }
        switch state {
        case .connected:
            if match.expectedPlayerCount == 0 { self.beginDataExchange() }
        case .disconnected:
            self.matchPhase = self.matchPhase == .playing ? .finished : .error("Your friend left.")
        default: break
        }
    }
}
```

`startCountdown()` in both manager files should also switch from `Timer.scheduledTimer + DispatchQueue.main.async` to a `Task { @MainActor in }` loop:
```swift
private func startCountdown() {
    matchPhase = .countdown(3)
    Task { @MainActor in
        for count in stride(from: 3, through: 1, by: -1) {
            try? await Task.sleep(for: .seconds(1))
            matchPhase = .countdown(count - 1)
        }
        try? await Task.sleep(for: .seconds(1))
        matchPhase = .playing
    }
}
```

`AllergenPickerStep.swift` `AllergenToggleButton` uses raw `.white` for text and icon when selected (two sites). ### SR-3: Centralize the hand-rolled CTA debt with a "GameCTA" button style

The 30+ hand-rolled CTAs in mini-games, versus screens, and completion views all share a common shape: a label with horizontal/vertical padding, a background color, and a cornerRadius. None need the wood-grain texture of `.texturedButton`, but they do need consistent sizing and the BouncyButtonStyle scale animation.

A lightweight `GameCTAStyle` that wraps `BouncyButtonStyle` with a capsule/rounded-rect background would let every game-screen CTA migrate in one sweep:
```swift
// Usage at every game-screen CTA:
Button("Play Again!") { startGame() }
    .buttonStyle(GameCTAStyle(tint: Color.AppTheme.sage))

// Rough implementation in AppTheme.swift:
struct GameCTAStyle: ButtonStyle {
    let tint: Color
    func makeBody(configuration: Configuration) -> some View {
        configuration.label
            .font(.AppTheme.headline)
            .foregroundColor(Color.AppTheme.cream)
            .padding(.horizontal, AppSpacing.lg)
            .padding(.vertical, AppSpacing.sm)
            .background(tint)
            .cornerRadius(AppSpacing.cardCornerRadius)
            .scaleEffect(configuration.isPressed ? 0.95 : 1.0)
            .animation(AnimationConstants.springQuick, value: configuration.isPressed)
    }
}
```

### SR-4: Fix the try? save pattern with a one-line helper

Twelve files need the same wrapping pattern. A `ModelContext` extension removes the boilerplate:
```swift
extension ModelContext {
    func saveWithLogging(file: String = #fileID) {
        do { try save() } catch { print("[\(file)] save failed: \(error)") }
    }
}
```
Then all call sites become `modelContext.saveWithLogging()` — no try-catch boilerplate, no accidental `try?` reversion.

## Section 8 — What's Working Well

1. **PipAIService.swift** — Textbook async/await architecture. `await MainActor.run { }` used correctly throughout the streaming loop, per-profile rate limiting scoped by `UUID`, graceful degradation between on-device (iOS 26) and cloud (Claude). The 2-hop tool-use loop with `cache_control: ephemeral` on the system prompt is particularly clean. Nothing to fix here.

2. **Weather + season token migration** — `WeatherOverlayView.swift` is fully migrated to named tokens (`Color.AppTheme.rainBlue`, `sunYellow`, `weatherCloudy`, `frostBlue`, `springPetal`, `autumnBrown`). Zero raw colors remain. The seasonal gradient arrays in `WeatherOverlayView` and `SeasonalOverlayView` read cleanly against the STYLES.md reference table.
