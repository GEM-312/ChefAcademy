## TL;DR

Three systemic issues dominate this audit:

1. **Silent free tier (P0):** `PipVoiceMode` has no free audio option — Apple TTS was removed. Free 6-year-olds who can't read receive zero voice guidance from Pip. For the stated 6+ audience this is a critical barrier.
2. **Hidden primary actions (P0/P1):** Two key CTAs are buried below un-cued scroll areas — "Let's Cook!" in `RecipeDetailView` and the Pip speech bubble in `PipAnimations`'s `PipWithDialogue` component (which unlike `PipSpeechBubble` never calls `speak()`).
3. **Affordance gaps on primary action buttons (P1):** Buttons throughout the social/recipe flows (RecipeDetailView, SiblingProfileView, ProfilePickerView, RecipeCardView) use `.buttonStyle(.plain)` or `.onTapGesture` — no bounce feedback for 6-year-olds who need explicit press confirmation.

The rest of the codebase is in notably good shape: allergen safety is thorough, `BouncyButtonStyle` is used consistently in mini-games, `PipSpeechBubble` auto-speaks, the morph transition in `PlantingSheet` is delightful, and the social garden-helping features are thoughtfully executed.

## P0 Blockers

### P0-1 — No free voice: 6-year-olds who can't read get nothing
**File:** `PipVoice.swift:32–49`

`PipVoiceMode` only has `.readText` (silent) and `.elevenLabs` (requires $3.99/month Pip Plus). Apple TTS was intentionally removed. The comment says "Better to have NO voice than a bad voice" — but for a child who cannot yet read, zero voice means zero Pip guidance.

```swift
enum PipVoiceMode: String, CaseIterable {
    case readText = "silent"        // No voice — kid reads on screen
    case elevenLabs = "elevenlabs"  // Premium Pip voice (subscription)
}
```

**Impact:** Every call to `PipVoice.shared.speak()` (across 20+ files) is silent for free users. NPC greetings in `PlantingSheet`, cooking step narration, `PipSpeechBubble` auto-speak, `PipHeaderStack` onboarding narration — all silent. This blocks the app from being usable by pre-readers without a subscription.

**Fix (tokens only):** Restore `.appleTTS` as a third case between `.readText` and `.elevenLabs`. Map it to the existing `speakWithAppleTTS()` method that is already implemented but unreachable (`PipVoice.swift:200–219`). In `VoicePickerView` add a third `VoiceOptionCard` for "Pip's Voice (Free)" with `color: Color.AppTheme.sage`. Update the `speak()` router at `PipVoice.swift:158–169` to handle the new case.

---

### P0-2 — `PipWithDialogue` never speaks
**File:** `PipAnimations.swift` — `PipWithDialogue.body` `.onAppear` block

`PipSpeechBubble` (`PipComponents.swift:76–81`) auto-speaks via `speakOnAppear: true`. `PipWithDialogue` — the animated component used in `HomeAnimated.swift`, `BodyBuddyView.swift`, and other screens — has an `.onAppear` that only triggers the animation delay; it never calls `PipVoice.shared.speak(message)`.

```swift
// PipAnimations.swift — PipWithDialogue.onAppear
.onAppear {
    // Missing: PipVoice.shared.speak(message)
    DispatchQueue.main.asyncAfter(deadline: .now() + 0.5) {
        withAnimation(.easeOut(duration: 0.3)) { showMessage = true }
    }
}
```

**Impact:** Pip appears to talk (animated mouth) but produces no audio. Parents/kids who have voice enabled will hear nothing on any screen that uses `PipWithDialogue` instead of `PipSpeechBubble`.

**Fix:** Add `PipVoice.shared.speak(message)` inside the `.onAppear` block (before the animation delay) and an `.onChange(of: message)` handler — matching `PipSpeechBubble`'s implementation exactly. Also add a `speakOnAppear: Bool = true` parameter so call sites can opt out for decorative usage.

### P0-3 — "Let's Cook!" CTA buried below un-cued scroll
**File:** `RecipeDetailView.swift:20,269–282`

The `ScrollView(showsIndicators: false)` hides the scroll indicator. "Let's Cook!" is the last item, placed after: hero image → title block → Pip's tip → garden ingredients → pantry ingredients → cooking steps → nutrition facts. A 6-year-old sees the hero image and recipe name, then hits the steps, and has no signal that a big green button exists below. `showsIndicators: false` removes the only visual cue that there's more to scroll.

```swift
// RecipeDetailView.swift:20
ScrollView(showsIndicators: false) {   // no scroll indicator
    ...
    // RecipeDetailView.swift:269 — only visible after scrolling past everything
    Button(action: { ... }) {
        Text("Let's Cook!")
    }
    .buttonStyle(.plain)               // no bounce feedback
```

**Fix:** Remove `showsIndicators: false` (or replace with `showsIndicators: true`). Extract "Let's Cook!" into a sticky footer outside the ScrollView so it is always visible. Use `.texturedButton(tint: Color.AppTheme.sage)` instead of the custom inline style to get bounce + consistent height.

## P1 Friction

### P1-1 — Onboarding dialog count exceeds 4-screen cap
**Files:** `MeetPipViews.swift`, `MeetPipAnimated.swift`, `FamilySetupView.swift`

### P1-2 — "Buy & Plant" / shop purchases have no confirmation
**Files:** `PlantingSheet.swift:282–292`, `FarmShopView.swift`

"Buy & Plant" in `PlantingSheet` spends coins AND plants the seed in a single tap. `FarmShopView` buy buttons immediately deduct coins. Neither presents a confirmation step. A 6-year-old who taps accidentally loses coins with no recovery path (Anti-pattern I).

```swift
// PlantingSheet.swift:283
Button("Buy & Plant \(veg.displayName)") {
    if gameState.buySeed(veg) {          // coins deducted immediately
        if let seed = ... { plantSeed(seed) }  // also planted immediately
    }
}
```

**Fix:** Wrap `buySeed` + `plantSeed` in a two-step flow: show a brief confirmation with `PipDialogView(message: "Spend \(veg.seedCost) coins and plant?", choices: ["Yes!", "Not yet"])`. Use `AnimationConstants.springBouncy` for the confirmation appearance.

### P1-3 — "Insulin Tetris" and medical vocabulary in Play tab
**Files:** `PlayLearnView.swift`, `InsulinTetrisView.swift`, `GlucoseJourneyView.swift`

`PlayLearnView` displays "Insulin Tetris" as a `MiniGameCard` title. `InsulinTetrisView` uses "Insulin", "Liver", "Fat cells", "fat is the last resort". `GlucoseJourneyView` uses "mitochondria" and "antioxidants" in body copy. These are adult endocrinology terms inappropriate for age 6+ (Anti-pattern F).

**Fix:** Rename "Insulin Tetris" → "Sugar Sorter" or "Energy Blocks". Replace body copy using `Color.AppTheme.darkBrown` text with kid-friendly language. Existing `PipAIService` system prompt correctly bans "glucose" and "insulin" in AI responses — apply the same ban to static UI strings.

### P1-6 — Food labels at 9pt during split-screen gameplay
**File:** `SplitScreenVersusView.swift:444`

Food name labels inside each player's game half render at `font(.AppTheme.rounded(size: 9, weight: .bold))`. The HIG minimum readable text is 11pt. During fast-moving gameplay, a 6-year-old reading "Candy Bar" vs "Avocado" at 9pt in a rotated, half-screen area is impractical.

```swift
// SplitScreenVersusView.swift:444
Text(item.food.name)
    .font(.AppTheme.rounded(size: 9, weight: .bold))  // too small
```

The same issue appears in `MultiplayerHealthyPicksView.swift` and `NearbyVersusView.swift` food labels at `size: 11`.

**Fix:** Raise to `font(.AppTheme.caption)` (the smallest AppTheme text token, which is 12pt). For split-screen where vertical space is limited, hide the text label entirely and rely on the food emoji + image — kids recognize visual icons faster than text during fast gameplay.

### P1-7 — Gift veggie is immediate with no confirmation
**File:** `SiblingProfileView.swift:252–292`, `GiftVeggieSheet`

Tapping a veggie in `GiftVeggieSheet` immediately calls `onGift(item.type)` → `giftVeggie()` which removes the veggie from inventory and calls `saveToStore()`. There is no "Are you sure?" step. Gifting is irreversible. (Anti-pattern I)

**Fix:** Add a `PipDialogView` confirmation before gifting: `"Give your \(vegType.displayName) to \(sibling.name)? You can't get it back!"` with choices `["Yes, gift it!"]` + `["Maybe not"]`. Use `AnimationConstants.springBouncy`.

## P2 Polish

### P2-1 — Nutrient row labels use `.rawValue` directly
**File:** `SeedInfoView.swift:809`

```swift
Text(nutrient.rawValue)
```

`NutrientType.rawValue` exposes labels like `"antioxidants"`, `"vitaminK"`, `"probiotics"` — adult biochemistry terms. The emoji, organ description, and USDA superpower text that surround it are kid-friendly, but the header label is not.

**Note:** The `SeedInfoView` comment explicitly explains that the color→pigment tip (Lycopene, Beta-carotene) is intentional science education. Apply the same logic here — but render "Vitamin K" as "Vitamin K (Healer)" or use the existing `organIcon` + `benefitsOrgan` properties as the primary label instead of `rawValue`.

**Fix:** Replace `Text(nutrient.rawValue)` with a kid-friendly display name. Add a `var kidFriendlyName: String` property to `NutrientType` using existing `AppTheme` font: `.AppTheme.rounded(size: 16, weight: .semibold)` (already used on line 810).

### P2-3 — `SeedInfoView` Pip color tip doesn't use `PipSpeechBubble`
**File:** `SeedInfoView.swift:564–597`

The Pip color tip (`pipColorTip`) is a manual `HStack` with `Image(PipPose.gotIdea.rawValue)` + a custom speech bubble. It doesn't call `PipVoice.shared.speak()` when the color changes. By contrast, `PipSpeechBubble` auto-speaks on appear and on `.onChange(of: message)`.

**Fix:** Replace the manual layout with `PipSpeechBubble(message: detectedColorChoice.pipTip, pose: .gotIdea, size: .medium, speakOnAppear: true)`. The `onChange` observation for color changes will then automatically call `speak()`.

### P2-9 — `PlantingSheet` close button appears before NPC is ready
